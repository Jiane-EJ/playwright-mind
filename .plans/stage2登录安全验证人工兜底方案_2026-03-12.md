# Stage2 登录安全验证自动处理方案（2026-03-12）

## 背景

执行 `stage2:run:headed` 时，登录阶段会不定期出现阿里云滑块/安全验证，导致后续菜单点击步骤在未进入首页时失败。

## 目标

实现滑块验证码的自动处理，同时保留人工兜底能力：

1. 自动检测阻塞性安全验证
2. **使用 AI + Playwright 自动完成滑块拖动**
3. 失败时回退到人工等待或明确报错
4. 超时时给出明确失败原因与配置建议

## 方案

1. 在 `src/stage2/task-runner.ts` 增加安全验证检测函数，基于常见文案和选择器识别滑块弹层。
2. 增加统一处理函数，支持四种模式：
   * `STAGE2_CAPTCHA_MODE=manual`：等待人工完成
   * `STAGE2_CAPTCHA_MODE=auto`：**AI 自动处理滑块**（默认）
   * `STAGE2_CAPTCHA_MODE=fail`：检测到即失败
   * `STAGE2_CAPTCHA_MODE=ignore`：忽略检测
3. **自动处理实现**：
   * 使用 Midscene `aiQuery` 分析截图获取滑块位置
   * 使用 Playwright `mouse` API 模拟真人拖动轨迹
   * 拖动算法：easeOut 缓动 + 随机抖动，模拟人手
4. 登录步骤后增加 `处理安全验证` 步骤；菜单点击前再次检查，避免登录后延迟弹出导致误执行。
5. 增加 `STAGE2_CAPTCHA_WAIT_TIMEOUT_MS` 控制人工等待超时。

## 自动处理流程

```
检测到滑块
    ↓
AI 分析截图获取滑块按钮坐标 (x, y)
    ↓
AI 分析截图获取滑槽宽度
    ↓
计算目标位置
    ↓
Playwright 模拟拖动（15步，easeOut缓动，随机抖动）
    ↓
检查滑块是否消失
    ↓
成功 → 继续执行
失败 → 重试（最多3次）
```

## 回滚

设置 `STAGE2_CAPTCHA_MODE=manual` 恢复人工兜底模式，或移除 `task-runner.ts` 新增的安全验证处理逻辑。
