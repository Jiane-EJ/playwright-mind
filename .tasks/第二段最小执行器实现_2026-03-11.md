# 第二段最小执行器实现

## 实施日期

2026-03-11

## 目标

在不开发第一段页面探索流程的前提下，先落地第二段最小可执行链路，实现：

* 从任务 JSON 读取执行参数
* 使用 Midscene + Playwright 自动执行业务步骤
* 生成结构化执行结果和步骤截图

## 变更内容

* 新增 `src/stage2/types.ts`：第二段任务模型与结果模型
* 新增 `src/stage2/task-loader.ts`：
  * 任务文件加载
  * 最小字段校验
  * 变量占位符解析（`${NOW_YYYYMMDDHHMMSS}`、`${ENV_VAR}`）
* 新增 `src/stage2/task-runner.ts`：
  * 登录、菜单导航、弹窗填写、提交、回查、断言流程编排
  * 步骤级状态记录与失败截图
  * 输出 `result.json`
  * 支持 `STAGE2_REQUIRE_APPROVAL=true` 时的人工审批门禁
* 新增 `tests/generated/stage2-acceptance-runner.spec.ts`：第二段执行入口测试
* 更新 `config/runtime-path.ts`：新增 `ACCEPTANCE_RESULT_DIR`
* 更新 `.env` / `.env.example`：
  * `ACCEPTANCE_RESULT_DIR`
  * `STAGE2_TASK_FILE`
  * `STAGE2_REQUIRE_APPROVAL`
* 更新 `package.json`：
  * `stage2:run`
  * `stage2:run:headed`
* 更新 `.gitignore`：忽略 `t_acceptance-results/`
* 更新 `README.md`：补充第二段运行方式与产物说明

## 运行方式

```shell
npm run stage2:run:headed
```

默认读取 `.env` 中 `STAGE2_TASK_FILE` 指向的任务文件。

## 产物目录

* `t_playwright-report/`
* `t_midscene_run/report/`
* `t_acceptance-results/<taskId>/<timestamp>/result.json`
* `t_acceptance-results/<taskId>/<timestamp>/screenshots/`

## 风险说明

* 当前为最小可执行版本，步骤通过自然语言驱动 Midscene，依赖页面文案稳定性
* 列表断言仍有一定 AI 识别波动，后续建议增加更多 `aiQuery + 代码硬断言`
* 还未接入第一段“探索 -> 人工复核 -> 生成任务”流程
