# 第一段自然语言请求转JSON

## 实施日期

2026-03-13

## 目标

解决第一段“必须手写 JSON 请求文件”的使用门槛问题，支持用自然语言快速生成可执行请求文件。

## 本次变更

  * 自动补齐第一段探索深度配置（`scope.explorationDepth=deep`）
  * 自动补齐第一段交互目标（`scope.interactionTargets`：按钮、链接、日期、单选、多选、下拉、级联等）
  * 输出默认文件：`specs/stage1/stage1-request.generated.json`
  * 支持 `--output` 指定输出路径、`--force` 覆盖写入

## 使用方式

1. 一句话生成请求：
   编辑 `specs/stage1/stage1-brief.txt`，再执行 `npm run stage1:generate-request -- --force`
2. 文件生成请求：
   `npm run stage1:generate-request -- --file specs/stage1/your-brief.txt --output specs/stage1/stage1-request.generated.json --force`
3. 将 `.env` 的 `STAGE1_REQUEST_FILE` 指向生成文件，再执行：
   `npm run stage1:run:headed`

## 边界说明

* 当前已支持：一句话文本、`.txt`、`.html/.htm`
* 当前未支持：`.doc/.docx` 结构化解析（需后续补充文档解析能力）
* 当输入文本中缺少 URL 或账号密码时，脚本会直接报错并提示缺失项
