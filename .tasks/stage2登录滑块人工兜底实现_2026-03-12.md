# Stage2 登录滑块自动处理实现

## 实施日期

2026-03-12

## 目标

解决 `stage2:run:headed` 在登录页偶发滑块/安全验证时，后续菜单点击直接失败的问题。

## 变更内容

* 更新 `src/stage2/task-runner.ts`：
  * 新增滑块/安全验证检测（常见文案 + 常见选择器）
  * 新增 `handleCaptchaChallengeIfNeeded` 统一处理逻辑
  * **新增 `auto` 自动模式**：使用 Midscene AI 分析滑块位置 + Playwright 模拟拖动
  * 新增 `querySliderPosition` 函数：AI 识别滑块按钮坐标
  * 新增 `querySliderTrackWidth` 函数：AI 识别滑槽宽度
  * 新增 `autoSolveSliderCaptcha` 函数：模拟真人拖动轨迹（先快后慢 + 随机抖动）
  * 登录后新增步骤：`处理安全验证`
  * 菜单点击前增加二次检查，防止延迟弹窗
* 更新 `.env` / `.env.example`：
  * `STAGE2_CAPTCHA_MODE=auto` (支持 manual/auto/fail/ignore)
  * `STAGE2_CAPTCHA_WAIT_TIMEOUT_MS=120000`

## 安全验证模式说明

| 模式 | 说明 |
|------|------|
| `manual` | 检测到滑块后等待人工完成（默认） |
| `auto` | 使用 AI + Playwright 自动处理滑块 |
| `fail` | 检测到滑块直接失败 |
| `ignore` | 忽略滑块检测 |

## 自动处理流程

1. 使用 `aiQuery` 获取滑块按钮位置 (x, y, width, height)
2. 使用 `aiQuery` 获取滑槽宽度
3. 计算目标位置（滑槽最右侧）
4. 使用 Playwright `mouse.move/down/up` 模拟拖动
5. 拖动轨迹模拟真人：easeOut 缓动 + 随机抖动
6. 验证滑块是否消失，最多重试 3 次
