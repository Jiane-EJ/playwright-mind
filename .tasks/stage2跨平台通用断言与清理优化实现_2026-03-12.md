# Stage2 跨平台通用断言与清理优化实现

## 实施日期

2026-03-12

## 目标

在不破坏现有任务 JSON 兼容性的前提下，提升多平台场景下的断言稳定性与数据清理安全性。

## 变更内容

* 更新 `src/stage2/types.ts`：
  * 新增 `TaskUiProfile`（可配置表格/提示/弹窗选择器）
  * `TaskAssertion` 增加 `matchMode`、`expectedColumnFromFields`、`expectedColumnValues`
  * `TaskCleanup` 增加 `rowMatchMode`、`verifyAfterCleanup`
* 更新 `src/stage2/task-runner.ts`：
  * 新增通用选择器解析与行匹配模式解析
  * `table-row-exists` 增加空值防御，支持 `exact/contains`
  * `table-cell-equals` 改为结构化列值比对，避免“仅匹配到行即通过”
  * `table-cell-contains` 改为依据返回 `cellValue` 做代码级包含判断
  * 数据清理增加目标去重、精确匹配、删除后行消失校验
* 更新任务模板与示例：
  * `specs/tasks/acceptance-task.template.json`
  * `specs/tasks/acceptance-task.community-create.example.json`
* 更新文档：
  * `README.md` 增加跨平台通用配置说明与推荐实践

## 补充收敛（同日）

* 更新 `src/stage2/task-runner.ts`：
  * `table-cell-equals` 增加 Playwright 表格列值提取与代码级精确比对，失败时输出缺列/期望值/实际值明细
  * `table-cell-contains` 增加 Playwright 表格列值提取与代码级包含判断
  * `executeAssertionWithRetry` 保留最后一次结果，便于失败时输出诊断信息
  * “检查提交提示”步骤改为基于真实检测结果决定是否跳过，修复未检测到仍记为 `passed` 的假通过问题
  * `STAGE2_CAPTCHA_MODE` 默认值与 `README.md` / `.env.example` 对齐为 `auto`

## 验证

当前会话执行环境缺少可用的 Node/Playwright 命令入口，未在本会话内完成自动化运行验证。
