# AGENTS

## 目标

统一项目内开发、命名、日志、配置和运行产物目录规范，降低多人协作和后续维护成本。

## 通用要求

* 统一使用中文交流
* 作者统一为：`@author Jiane`
* 优先复用项目现有实现方式、工具类和目录结构
* 配置、路径、开关项优先通过 `.env` 管理，禁止在业务代码中散落硬编码

## 命名规范

* 包名、目录名使用小写或约定俗成的短横线风格
* 类名、类型名使用大驼峰
* 变量名、方法名使用小驼峰
* 常量使用全大写加下划线
* 运行时自动生成目录统一以 `t_` 开头

## 配置规范

* 所有环境变量必须同步维护 `.env.example`
* 新增配置时，优先提供默认值和用途说明
运行产物目录统一通过以下环境变量管理：
* `RUNTIME_DIR_PREFIX`
* `PLAYWRIGHT_OUTPUT_DIR`
* `PLAYWRIGHT_HTML_REPORT_DIR`
* `MIDSCENE_RUN_DIR`
* 涉及路径拼装时，统一通过公共配置模块处理，避免各处重复计算

## 日志规范

* 保留必要的执行日志、错误日志和测试产物日志
* 错误日志必须保留完整异常堆栈，禁止吞异常
* 日志内容应包含足够上下文，避免只有结果没有原因
* 运行日志、缓存、报告统一落到配置指定目录，不直接写死目录名

## 自动生成目录规范

* Playwright 执行产物目录默认使用 `t_runtime/test-results`
* Playwright HTML 报告目录默认使用 `t_runtime/playwright-report`
* Midscene 运行根目录默认使用 `t_runtime/midscene_run`
* 新增自动生成目录时，必须接入 `.env`，并保持 `t_` 前缀规则
* `.gitignore`、CI 路径、README 说明必须同步更新

## 测试与变更要求

* 修改配置后至少执行一次基础校验，确认目录输出位置正确
* 涉及 CI 的目录调整，必须同步修改工作流上传路径
* 涉及文档或规范的调整，必须同步更新 README 或对应说明文件
* 开发记录统一写入 `.task/` 目录，文件名格式为 `任务名称_yyyy-MM-dd.md`

## 实施原则

* 小步修改，保持可回滚
* 优先兼容现有测试代码和执行方式
* 非必要不引入新依赖
* 新增公共能力优先抽取到独立配置模块，避免重复实现
